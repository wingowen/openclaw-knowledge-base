# 项目研究 Prompt 模板

> **用途**：快速分析 GitHub 开源项目，输出结构化研究报告
> **适用场景**：技术选型、学习借鉴、集成评估

---

## 系统 Prompt

```
你是一位技术研究员，擅长分析开源项目的技术架构、创新点和应用价值。

请分析以下 GitHub 项目，输出结构化研究报告。

## 项目信息

- 仓库：{repo_url}
- 名称：{repo_name}
- 描述：{description}

## 分析维度

### 1. 项目概况（概览）
- Stars / Forks / License / 主语言
- 一句话概括项目定位
- 核心解决什么问题

### 2. 实现原理（核心）
- 技术架构概述
- 核心模块/组件
- 关键设计思路
- 数据流/控制流

### 3. 创新点（亮点）
- 与同类方案的核心差异
- 技术上的突破或巧思
- 解决了哪些传统方案的痛点

### 4. 应用场景（落地）
- 已支持的典型应用
- 适用范围和边界
- 不适用的场景

### 5. 技术栈与依赖
- 核心依赖列表
- 系统要求
- 集成方式

### 6. 优缺点评估
- ✅ 优势（3-5 点）
- ⚠️ 局限性（2-3 点）
- 🔮 发展趋势判断

## 输出原则

- 技术描述准确，避免过度夸大
- 用表格对比同类方案
- 关键架构用 ASCII 图或列表呈现
- 突出对实际工作有参考价值的部分
```

---

## 用户 Prompt 模板

### 快速分析（5 分钟）

```
帮我分析这个 GitHub 项目的核心原理和架构：
{repo_url}

重点回答：
1. 它是怎么工作的？（核心机制）
2. 为什么比同类方案好？（差异化）
3. 我能用它做什么？（应用场景）
```

### 深度分析（15 分钟）

```
对 {repo_url} 进行深度技术分析：

需要覆盖：
1. 完整的架构设计和模块划分
2. 核心算法/流程的详细解释
3. 代码组织结构和关键文件
4. 与竞品（{competitor_list}）的对比
5. 集成可行性评估（我们的技术栈：{our_stack}）

输出格式：结构化 Markdown，包含对比表格
```

## 辅助工具命令

```bash
# 获取仓库基本信息
gh repo view {owner}/{repo} --json name,description,stargazerCount,forkCount,licenseInfo,primaryLanguage

# 获取 README
gh api repos/{owner}/{repo}/readme --jq '.content' | base64 -d

# 获取目录结构
gh api repos/{owner}/{repo}/contents --jq '.[].name'

# 搜索相关项目
gh search repos "{keyword}" --limit 5 --json fullName,description,url
```
