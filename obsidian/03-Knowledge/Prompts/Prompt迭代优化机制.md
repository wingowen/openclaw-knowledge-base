# Prompt 迭代优化机制

> **原则**：每次生成报告 = 一次 Prompt 实验。输出质量决定 Prompt 进化方向。
> **状态**：🔄 持续运行中

---

## 优化循环

```
┌──────────────────────────────────────┐
│  1. 使用当前 Prompt 生成报告          │
└──────────────┬───────────────────────┘
               │
               ▼
┌──────────────────────────────────────┐
│  2. 评估输出质量（自动 + 人工）        │
│  - 结构完整性                         │
│  - 数据准确性                         │
│  - 分析深度                           │
│  - 可操作性                           │
└──────────────┬───────────────────────┘
               │
               ▼
┌──────────────────────────────────────┐
│  3. 识别问题                         │
│  - 哪些章节空泛？                     │
│  - 哪些结论缺少数据支撑？              │
│  - 哪些预判后来被打脸？                │
│  - 哪些格式不符合预期？                │
└──────────────┬───────────────────────┘
               │
               ▼
┌──────────────────────────────────────┐
│  4. 修改 Prompt                       │
│  - 增加约束 / 明确格式                │
│  - 补充示例 / 调整权重                │
│  - 优化数据输入结构                    │
└──────────────┬───────────────────────┘
               │
               ▼
         回到步骤 1
```

---

## 评估维度与权重

| 维度 | 权重 | 评分标准 |
|------|------|----------|
| **结构完整性** | 20% | 8 章节齐全，每章有实质内容（非占位符） |
| **数据准确性** | 25% | 指数/个股数据与实际行情一致，涨跌方向正确 |
| **分析深度** | 25% | 有因果分析，不是简单复述数据 |
| **预判准确度** | 20% | 次日实际走势与预判吻合程度 |
| **可操作性** | 10% | 策略建议具体，有明确的仓位/方向/止损 |

**综合评分** = Σ(维度分 × 权重)，满分 10 分

## 问题 → Prompt 修改映射

### 常见问题与修复策略

| 问题表现 | 根因分析 | Prompt 修改 |
|----------|----------|-------------|
| 分析空泛，像在说废话 | 缺少数据约束 | 增加「必须引用具体数据」要求 |
| 预判模糊，左右都对 | 缺少明确判断 | 增加「必须给出明确方向」+「附置信度」 |
| 格式混乱，章节不全 | 输出约束不够 | 增加强制格式模板 + checklist |
| 量能分析缺失 | 数据输入不完整 | 补充成交额/量比数据到输入 |
| 情绪判断不准 | 缺少量化指标 | 增加涨跌分布/涨停数/连板数输入 |
| 策略建议不可操作 | 缺少具体化要求 | 要求明确写出「买什么/买多少/何时卖」 |

## 每次报告后的自检 Checklist

生成报告后，对照检查：

```
□ 8 个章节是否全部有实质内容？
□ 每个数据点是否标明来源和时间？
□ 预判是否给出了明确方向（不是"可能涨可能跌"）？
□ 策略部分是否包含具体仓位建议？
□ 风险提示是否具体（不是"注意风险"这种废话）？
□ 与昨日报告是否有连续性（复盘对比）？
```

**如果有任意一项为否 → 记录问题 → 优化 Prompt**
