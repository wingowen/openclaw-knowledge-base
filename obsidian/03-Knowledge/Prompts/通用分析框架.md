# 通用分析框架 Prompt

> **用途**：适用于各类技术/商业分析的通用框架
> **可扩展**：按需组合不同分析维度

---

## 信息收集框架

```
分析目标：{target}

第一层：基本信息
- 是什么？（定义/定位）
- 谁做的？（团队/组织）
- 什么时候？（时间线）
- 多大规模？（数据指标）

第二层：技术细节
- 怎么实现的？（核心技术）
- 用了什么？（技术栈）
- 架构怎样？（模块划分）
- 有何创新？（差异化）

第三层：价值评估
- 解决什么问题？（需求场景）
- 效果如何？（性能/效果指标）
- 有何局限？（边界/缺陷）
- 值得借鉴？（可复用点）

第四层：行动建议
- 能用在哪？（应用场景）
- 怎么集成？（接入方式）
- 风险点？（潜在问题）
- 下一步？（具体行动）
```

---

## 快速判断框架（30 秒）

```
🎯 核心价值：一句话说清楚它解决了什么问题
⚡ 关键差异：和现有方案比，最大的不同是什么
📊 成熟度：Stars / 更新频率 / 文档质量 / 生产可用
🎯 相关性：和我的需求/项目有多大关联
```

---

## 技术评估 Checklist

```
□ 文档完整性
  - README 是否清晰
  - 有无 Quick Start
  - API 文档是否完善
  - 有无中文文档

□ 代码质量
  - 目录结构是否合理
  - 测试覆盖率
  - CI/CD 配置
  - Issue/PR 活跃度

□ 生态与社区
  - Stars 趋势
  - 贡献者数量
  - 更新频率
  - 周边工具/插件

□ 生产就绪
  - 版本号（0.x = 实验性）
  - Breaking Change 频率
  - 性能基准
  - 安全审计
```
