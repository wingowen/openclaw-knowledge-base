## 🗃️ 二、数据层方案

### 2.1 数据库记录规划

| 资源 | 数量 | 说明 |
|------|------|------|
| 标签 | 1 | `新概念英语第二册` (color 建议用 #D2691E 巧克力色) |
| 文章 | 96 | 对应 96 课，每课一个 article 记录 |
| 句子 | ~1500 | 每课约 15-20 句，总计约 1500-2000 句 |
| 音频 | 可选 | 暂不自动生成，后续可集成 TTS |

### 2.2 标签创建 SQL

```sql
INSERT INTO tags (id, name, color, created_at, updated_at)
VALUES (7, '新概念英语第二册', '#D2691E', NOW(), NOW())
ON CONFLICT (id) DO UPDATE SET
  name = EXCLUDED.name,
  color = EXCLUDED.color;
```

**推荐 tag_id = 7**（第三册用了 6，第二册用 7，避免冲突）

### 2.3 文章记录格式

```json
{
  "title": "1. Lesson 1: Morning Coffee",
  "description": "新概念英语第二册 Lesson 1",
  "source_url": "https://newconceptenglish.com/index.php?id=nce-2-001",
  "total_sentences": 14,
  "created_at": "2026-03-12T...",
  "updated_at": "2026-03-12T..."
}
```

### 2.4 句子记录格式

```json
{
  "article_id": <关联的文章ID>,
  "content": "Do you want a cup of tea or coffee?",
  "sequence_order": 1,
  "is_active": true,
  "extensions": {
    "translation": "你想喝茶还是咖啡？"
  }
}
```

## ⚠️ 六、风险与注意事项

### 6.1 数据抓取风险

- **网站反爬**：newconceptenglish.com 可能对高频请求限制
  - **缓解**：在 `fetch_nce2_lesson.py` 中添加 `time.sleep(1)`
  - **降级**：单课手动抓取后保存 JSON，批量时读取本地

- **HTML 结构变化**：第二册的 DOM 结构与第三册可能不同
  - **缓解**：先手动访问 `nce-2-001` 和 `nce-2-096` 对比结构
  - **备选**：使用更通用的文本提取库（如 `readability-lxml`）

### 6.2 句子切分准确率

第三册的 `split_sentences()` 已针对 B.C./Mr. 等缩写优化，第二册的课文密度较低，预计准确率更高。

**验证方法**：抽样 10 课，人工检查切分结果。

### 6.3 数据库写入配额

Supabase 免费层：
- 写入峰值 2 req/s
- 每月 500 MB 数据

1500 条句子 × ~100 bytes ≈ 150 KB，完全无压力。

### 6.4 前端兼容性

`ImmersiveSpelling` 组件已修复 `useRef` 问题，但需重新验证第二册的句子长度不影响布局（如超长句子换行）。
